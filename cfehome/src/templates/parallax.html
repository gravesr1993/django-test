Parallax example

This utilizes a workaround due to conflicting problems within CSS and django I was currently unable to identify with position:fixed;
after extensive research, rather than fix the internal problem I found it easier to create a workaround with current knowledge,
to reflect an experience similar to a parallax effect in a more limit capacity.
This utilizes nth of type as well, something you might not have seen before, I would usually recommend against it but figured i'd include in this example
to demonstrate breadth of knowledge

<section class="parallax "><br> <p class="p-parallax sticky bg-dark text-white"><br> .parallax {
top: 0%;
/*backwards compat sticky*/
position: -webkit-sticky;
position:sticky;
background-attachment: local;
background-repeat: no-repeat;
background-size: cover;
height: 100%;
width: 100%;
margin-left: auto;
margin-right: auto;}

.parallax:nth-of-type(1) {
background: url('images/n1.jpg');}

.parallax:nth-of-type(2) {
background:url('images/n2.jpg');}

.parallax:nth-of-type(3) {
background:url('images/n3.jpg');}

.parallax:nth-of-type(4) {
background-color:lightgreen;}

This is a parallax view demo created with django specific implementation based off of workable CSS properties. Scroll Down to continue.

Scrolling on the page moves fixed view items, which in Django's case appears to not work. The workaround i've created utilizes sticky positions in lieu of fixed positioning.
